# Security Policy

## Scope / Supported Versions

Security support is provided for:

- the latest `main` branch
- the latest published release

Older versions may not receive security fixes.

## Reporting a Vulnerability

To report a potential vulnerability:

- open a GitHub Issue in this repository
- or use maintainer contact channels already listed in this repository/profile

Please include clear reproduction steps, affected version/commit, and impact details.

## Security Disclaimer

- This software is provided **as-is**.
- There has been **no formal security audit**.
- Users are responsible for secure key management and independent transaction verification.
- You use this software entirely at your own risk and are solely responsible for transaction outcomes and private key storage.
- Use low-balance or testing wallets first before handling significant funds.
